# 홈페이지 리디자인 — 화교 시장 진출 전략

작성일: 2026-04-29
목적: 베트남 화교(Hoa) 시장 테스트를 위한 dflo 홈페이지 리디자인. 한국어 / 중국어 공통 사용.

---

## 1. 전략 개요

### 시장
- **타겟**: 베트남 화교 (Hoa, 호치민 5군 Chợ Lớn 중심 80만명)
- **확장 근거**: 말레이시아 · 싱가포르 · 인도네시아에서 화교 시장 검증 완료
- **언어**: 간체 중국어 + 베트남어 부제 병기 (광동·조주·만다린 멀티 언어)
- **상담 채널**: WhatsApp · Zalo · WeChat (UTM 분리)

### 펀넬
```
FB 광고 → 화교용 홈페이지 → 4가지 기능 노출 → 상담 버튼 클릭
                                                  ↓
                                   [원격 상담 (목표 컨버전)]
```

### 핵심 KPI
- 첫 방문 상담 클릭률: **1-3%** (현실 벤치마크)
- 리타겟팅 누적: **5-10%**
- 합격선: **MY/SG/ID 동일 캠페인 KPI ±30% 이내**

## 2. 광고 전략 (Cold → 사이트 유입)

### 채널 비중 (1차 테스트)

| 채널 | CPM | 비중 |
|---|---|---|
| FB+IG Reels | 가장 저렴 | **60%** ⭐ |
| FB 피드 비디오 | 중간 | 25% |
| FB 캐러셀 (케이스용) | 중간 | 10% |
| TikTok (2주차 검증) | 저렴 | 5% |

### 광고 Angle (사이트 4가지 feature 기반)

| Angle | feature | 적합도 | 단계 |
|---|---|---|---|
| **A. 예측키 측정 후크** ⭐ | 1. 예측키 측정 | 매우 높음 | 메인 광고 entry |
| **B. 케이스 변화** | 3. 치료 사례 | 높음 | 사회증명 |
| **C. 가이드 요약** | 2. 성장 가이드 | 중간 | 보조 |
| **D. 원격 상담 강조** | 4. 원격 상담 (유료) | 낮음 | **리타겟팅 only** |

→ **예측키 측정**이 광고 entry로 가장 강함. 진입장벽 0, 즉시 가치 제공, 사람 응대 불필요, pixel 데이터 풍부.

### Reels 5종 1차 테스트 구성
- A1, A2: 예측키 측정 hook 2종
- B1: 케이스 디테일
- C1: 가이드 요약
- 같은 본문 + 첫 2초만 다른 hook 변형으로 빠른 학습

### 광고 vs 오가닉 분리

| 요소 | 광고용 | 오가닉용 |
|---|---|---|
| Hook | 강한 후크 (충격·궁금증) | 일상적·진심 |
| 길이 | 15-25s | 30-60s |
| 편집 | 빠른 컷, 모션 텍스트 | 단순, 인간적 |
| CTA | 명확·즉각 | 약하거나 없음 |

→ 같은 소스 영상으로 cut을 따로 만들기. 같은 cut을 양쪽에 쓰면 둘 다 어정쩡해짐.

## 3. 사이트 UX/UI 방향

### 진단

현재 사이트 = **콘텐츠 사이트 (인스타 카드 톤)**로 잘 만들어짐. 하지만 **광고 cold traffic 컨버전 랜딩**으로는 약함.

| 문제 | 원인 |
|---|---|
| 트러스트 시그널 밀도 부족 | 4:5 카드 한 장에 6개 시그널 못 담음 |
| FAQ·비교표·정보 밀도 표현 한계 | 모든 섹션이 슬라이드 캐러셀 템플릿 |
| 의료 진중함과 톤 미스매치 | 인스타 캐주얼 = 라이프스타일 |

### 결정: 카드 톤 유지 + 새 섹션 타입 추가

**모바일 위주 + 카드 톤 유지** ✓ — 다만 **모든 섹션이 캐러셀일 필요는 없음**.

| 패턴 | 적용 섹션 | 이유 |
|---|---|---|
| **info-stack** (신규) | Hero, 병원 소개, FAQ, CTA 마무리 | 정보 밀집 한 화면 노출 |
| **카드 캐러셀** (현재 유지) | 치료 프로그램, 치료 사례, 가이드 | 동등한 다수 옵션, 디테일 콘텐츠 |

→ info-stack = "그냥 평범한 vertical 섹션" (캐러셀 없이 요소들이 vertical로 stack). 새 개념 아님, 보통 웹사이트의 표준 구조.

## 4. 새 섹션 구조 (제안)

```
[1. Hero]               info-stack    예측키 측정 직접 임베드 + CTA + 트러스트 3개
       ↓
[2. 병원 소개]          info-stack    원장 + 통계 + 차별점 + 미디어 + 클리닉 사진
       ↓
[3. 치료 프로그램]      카드 캐러셀  ← 현재 그대로 (5개 카드 + iframe 모달)
       ↓
[4. 치료 사례]          카드 캐러셀  ← 현재 그대로
       ↓
[5. FAQ]                info-stack    5개 핵심 질문 아코디언
       ↓
[6. 성장 가이드]        카드 캐러셀  ← 현재 그대로 (13개)
       ↓
[7. CTA 마무리]         info-stack    최종 상담 유도
       ↓
[하단 sticky 탭바]      예측키 측정 / 무료 상담
```

## 5. 섹션별 상세 설계

### 5.1 Hero (info-stack)

**목표**: 광고로 들어온 부모가 즉시 측정 도구 사용. above-the-fold에 모든 행동 유도 요소 노출.

```
┌──────────────────────────────────────┐
│  韩国 187 成长诊所 · 30秒免费测试    │ ← 1초 컨텍스트
│                                       │
│  您孩子18岁能长多高?                  │ ← 큰 후크
│  父母身高填一下,马上知道              │
│                                       │
│  [성별: 남 / 여]                     │ ← 입력 1
│  [아빠 키: ___ cm]                   │ ← 입력 2
│  [엄마 키: ___ cm]                   │ ← 입력 3
│                                       │
│  [    🔮 예측 결과 보기    ]         │ ← 큰 CTA
│                                       │
│  ✓ 已为 3,000+ 家庭服务              │ ← 트러스트 1
│  ✓ 中文 / 越南文 在线咨询            │ ← 트러스트 2
│  ✓ 韩国儿科专医                      │ ← 트러스트 3
└──────────────────────────────────────┘
```

### 5.2 측정 결과 페이지 — 강한 전환 지점

**핵심 인사이트**: 대부분이 평균치 예측 → 평균을 만족하는 부모는 없음 → 자동 컨버전 부스트

```
您孩子的预测身高: 175 cm
   = 평균 50 백분위수 (또래 중간)

그런데 — 같은 부모키에서도
실제로 178~182cm 까지 큰 아이들이 있습니다.

[차트: 50백분위 선 + 187 환자 실제 도달 점]

차이는 무엇이었을까요?
[무료 첫 상담으로 확인하기 →]
```

**카피 원칙**: 평균을 부드럽게 포장하지 말고 정직하게. **"같은 조건에서 더 큰 아이가 있다"** 가 부모 머릿속을 흔드는 핵심.

### 5.3 병원 소개 (info-stack) — 7 블록

mockup: `v4/public/mockups/clinic-intro.html`

1. **섹션 헤드라인** — "한국 187 키 성장 클리닉" + 한 줄 가치 제안
2. **원장 프로필 카드** — 사진 + 자격 (소아청소년과 전문의 / 15년+ / 대한소아과학회)
3. **숫자 통계 4분할** — 3,000+ 가족 / 15+ 경력 / 240+ 사례 / 12+ 화교
4. **차별점 4가지** — AI 키 예측 / 다국어 상담 / 원격진료 / 골연령 분석
5. **미디어 노출** — 한국 방송·신문 로고 그리드
6. **클리닉 사진** — 진료실 + 외관 (강남구 명시)
7. **Inline CTA** — 그라데이션 버튼

**채워넣을 placeholder**: 원장 사진, 클리닉 사진, 정확한 숫자 (특히 화교 가족 수), 미디어 로고

### 5.4 치료 프로그램 — 현재 그대로

- 5개 프로그램 짧은 카드 (vertical stack)
- 클릭 시 → 풀스크린 모달 (iframe으로 `/programs/[name].html` 로드)
- 기존 인프라 그대로 재사용 (CLAUDE.md "Program HTML pages: v4/public/programs/")
- **변경**: 긴 이미지 → HTML 변환 (다국어 처리, 로딩 속도, 모바일 reflow, SEO 위해)
- 모달 안 프로그램 페이지 끝에 상담 CTA 박기

### 5.5 FAQ (info-stack) — 5개 핵심 질문

mockup: `v4/public/mockups/faq.html`

| # | 질문 | 해소하는 의심 |
|---|---|---|
| Q1 | 한국 안 가도 진료 가능? | 가장 큰 진입 장벽 |
| Q2 | 비용? | 결정 핵심 변수 |
| Q3 | 중국어/베트남어 OK? | 응대 신뢰도 |
| Q4 | 해외 화교 진료 경험? | 사회증명 |
| Q5 | 베트남에서 약 받을 수 있나? | 실용적 fulfillment |

**디자인**:
- 카드형 아코디언 (펼치면 primary 색 보더로 강조)
- 답변 첫 줄에 굵은 강조 결론 ("네, 가능합니다")
- 마지막에 inline CTA

**별도 FAQ 페이지 X** — 의심은 스크롤 흐름 안에서 풀려야 함. 별도 페이지는 "의심 깊은 사람"만 이동 = 이미 다 떠난 후.

### 5.6 CTA 전략 (3-layer)

| 위치 | 역할 |
|---|---|
| **하단 sticky 탭바** | 안전망, 지속 노출, 항상 시야 |
| **섹션 끝 inline CTA** | 감정 절정에서 클릭 유도 (특히 케이스·가이드 직후) |
| **측정 결과 페이지** | commitment escalation, 가장 강한 전환 |

세 layer가 보완 관계라 중복 아님.

**헤더에 상담 버튼 추가는 비추** — 모바일 헤더 좁아 시각적 부담, 하단 탭바와 기능 중복.

## 6. 기술 변경

### 6.1 새 슬라이드 템플릿: `IframeSlide`

목적: info-stack 섹션 (Hero · 병원 소개 · FAQ · CTA)을 정적 HTML 파일로 운영.

```ts
// v4/src/features/website/types/websiteSection.ts
export interface IframeSlide {
  template: 'iframe';
  id: string;
  src: string;              // /mockups/clinic-intro.html 등
  height?: number;          // px (고정), 미지정시 100vh 또는 auto
  showFrame?: boolean;      // 프레임 보이기/안보이기 (default: false)
  zoom?: number;            // 콘텐츠 zoom % (default: 100)
  bgColor?: string;
  order: number;
}
```

`showFrame` 옵션:
- `false` (기본): 헤더·외곽 보더 없이 매끄럽게 섹션 안에 녹아듦 (info-stack 적합)
- `true`: 헤더 + 보더 + "전체화면 보기" 버튼 (긴 콘텐츠 강조 시)

### 6.2 수정할 파일

1. `v4/src/features/website/types/websiteSection.ts` — `IframeSlide` 타입 + `Slide` union + `isIframeSlide` guard
2. `v4/src/features/website/components/SectionSlider.tsx` (또는 `SectionCarousel`) — iframe 렌더링 분기
3. `v4/src/features/website/pages/AdminBannerPage.tsx` — admin에 iframe 슬라이드 추가 폼

### 6.3 패턴 장점

1. **번역 = 새 HTML 파일만** (`*-zh.html`, `*-vi.html`)
2. **React 코드 안 건드림** — 정적 HTML만 수정
3. **info-stack / FAQ / Hero 변형 모두 같은 템플릿 1개로 처리**
4. **iframe-embeddable 인프라 그대로 재사용** — 검증된 패턴

### 6.4 추적 (FB Pixel + 이벤트)

- PageView (홈 도달)
- ViewContent (케이스 페이지 진입)
- ScrollDepth 50% / 75%
- Lead (상담 버튼 클릭) — 채널별 UTM 분리 (whatsapp / zalo / wechat)
- 측정 도구 사용 = 별도 이벤트 (리타겟팅의 핵심)

### 6.5 기술 점검

- **베트남 CDN 속도** — 이미지 webp + lazy load (이미 R2 사용 중이라 OK 추정)
- **모바일 우선** — 베트남 모바일 99%, 4G 평균 한국보다 느림
- **FB 광고 정책 점검** — 의료 결과 약속 ("키 X cm 성장!") 카피는 광고용으로 별도 작성, 구체 약속은 랜딩에서만

## 7. 운영

### 7.1 상담 응대 (테스트 단계)

**전략**: AI 번역 + 사람이 쓴 첫 응답 템플릿 + 30초 검수 (하이브리드)

**구성**:
1. **첫 응답 템플릿 5-10개** — 간체 중국어 네이티브가 한 번 잘 써둔 템플릿
   - 인사 + 클리닉 소개
   - "감사합니다, 잠시만요. 의사 확인 후 답변드릴게요"
   - 비용 안내
   - 한국 방문 절차
   - 핵심 FAQ
2. **AI 번역에 의료 용어집 + 톤 가이드 주입** — GPT/Claude API 시스템 프롬프트
3. **의사 답변 → AI 번역 → 사람 30초 검수** — MY/SG/ID 운영자 1명이 베트남 메시지 같이
4. **이탈 패턴 모니터링** — 첫 응답 후 24시간 내 재응답 없는 비율 추적

**도구 옵션**:
- WATI / Respond.io: WhatsApp + Zalo 통합 inbox + 자동 번역 + 템플릿
- 처음엔 WhatsApp Business 앱 + ChatGPT 옆에 띄워놓고 수동 복붙도 충분

**SLA**: 광고 클릭 후 메시지 응답까지 **30분 이내** 안 되면 전환 반토막.

**확장**: 시그널 검증되면 (1-2개월 뒤) 화교 응대 외주 1명 (필리핀/말레이 화교 프리랜서, 시간당 $5-10)

### 7.2 1차 결정 게이트

- **상담 클릭당 비용 MY/SG/ID 대비 ±30% 이내** → 본격 확장
- 그 이상 → 크리에이티브 / 타겟 재조정 후 2차 테스트

### 7.3 화교 부모 5가지 의심 (광고+사이트가 다 풀어야 함)

> "이 클리닉 진짜인가? · 화교 봐주나? · 비용? · 한국 가야 하나? · 다른 부모는 어땠나?"

- **이 클리닉 진짜?** → 병원 소개 섹션 + 미디어 노출
- **화교 봐주나?** → 차별점 (다국어) + Q3 + Q4 + 통계 (12+ 화교)
- **비용?** → Q2
- **한국 가야 하나?** → Q1 + 차별점 (원격 진료)
- **다른 부모는?** → 치료 사례 섹션 + Q4 (12+ 화교)
