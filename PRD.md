# 187 성장케어 - Product Requirements Document (PRD)

> Version 2.0 | 2026-03-24
> 연세새봄의원 187 성장 클리닉 — 소아 성장 관리 플랫폼 + 병원 웹사이트

---

## 1. 제품 개요

### 1.1 비전
소아 성장 관리의 디지털 전환 — 부모가 자녀의 성장을 체계적으로 추적하고, 병원과 연결되며, AI 기반 분석을 받을 수 있는 통합 플랫폼.

### 1.2 제품 구성

| 제품 | 대상 | URL | 인증 |
|------|------|-----|------|
| **환자 앱** | 등록 부모 | `/` | 이메일 로그인 |
| **관리자 앱** | 병원 직원 | `/admin/*` | 관리자 로그인 |
| **병원 웹사이트** | 일반 방문자 | `/website` | 없음 (공개) |
| **AI 서버** | 내부 | localhost:3001 | API Key |

### 1.3 핵심 지표 (KPI)
- 카카오톡 상담 문의 전환율 (웹사이트 → 카카오톡 클릭)
- 예상키 측정 사용률 (웹사이트 계산기 사용 수)
- 일일 활성 사용자 (환자 앱 DAU)
- AI 식단 분석 사용 빈도

## 2. 환자 앱 (v4)

### 2.1 사용자
- 연세새봄의원에 등록된 환자 부모
- 앱 로그인 필요 (이메일 + 비밀번호)

### 2.2 핵심 기능

#### 홈 (`/`)
- 선택된 자녀의 성장 요약 (최근 키/체중, 백분위)
- 콘텐츠 스와이프 카드 (성장 가이드, 레시피, 치료 사례)

#### 데일리 루틴 (`/routine`)
- 캘린더 기반 일일 기록
- 수면 기록 (취침/기상 시간, 메모)
- 식사 기록 (아침/점심/저녁) + AI 사진 분석
- 운동 기록 (세트/반복 + 유튜브 가이드)
- 영양제/주사 기록

#### 체형 분석 (`/body-analysis`)
- 체형 사진 업로드 → AI 자세 분석
- **현재 상태**: Mock (랜덤 데이터)
- **목표**: Gemini 2.5 Flash 실제 연동

#### 성장가이드 (`/info`)
- 성장 가이드 목록 + 상세
- 건강 레시피 목록 + 상세
- 치료 사례 목록 + 상세 (성장 차트 + 측정 기록)

### 2.3 AI 기능

| 기능 | 상태 | 기술 | 설명 |
|------|------|------|------|
| 식단 사진 분석 | ✅ 동작 | Gemini 2.5 Flash | 사진 → 메뉴 인식 → 영양 분석 → 성장 점수 |
| 체형 자세 분석 | ⚠️ Mock | (Gemini 예정) | 사진 → 자세 평가 → 교정 권장 |
| RAG 챗봇 | 🔜 미정 | (미정) | 성장 관련 질의응답 |

## 4. 병원 웹사이트 (Phase 7) ✅

### 4.1 목적
- 기존 Wix 사이트(yssaebomq.com) 리뉴얼
- **핵심 목표**: 카카오톡 상담 문의 전환율 극대화
- 방문자가 예상키 측정 → 콘텐츠 탐색 → 카카오톡 상담으로 이어지는 퍼널

### 4.2 페이지 구조

#### 메인 랜딩 (`/website`)
```
┌─────────────────────────────────────┐
│ Header: 로고 | 병원소개 | 프로그램 | 커뮤니티 | 상담예약 | ⚙️ │

│ [1] 롤링 히어로 배너 (4슬라이드, 5초 자동) │

│ [2] 신뢰 통계: 15,000+ | 94.7% | 20년+      │
├─────────────────────────────────────┤
│ [3] 예상키 측정 계산기                        │
│     성별, 생년월일, 현재키, 부모키 → 예상 성인키        │
```

#### 프로그램 상세 (`/website/program/:slug`)
- 7개 프로그램 각각 상세 페이지
- 원본 사이트 이미지 + 대상 아동 + 특징 + 유튜브 비디오

#### 배너 관리 (`/website/admin/banners`)
- PIN 보호 (8054)
- 슬라이드 CRUD: 제목, 부제목, CTA, 이미지/그라데이션

### 4.4 전환 최적화 요소
- 모든 섹션에 카카오톡 상담 CTA
- 플로팅 카카오톡 버튼 (항상 노출)
- 예상키 측정 후 "상담 받아보기" 유도
- 치료 사례의 구체적 성장 수치 (+32.7cm 등) 강조
- 신뢰 지표 (누적 치료 수, 성공률, 경력)

### 4.5 콘텐츠 소스
- **프로그램 데이터**: 원본 사이트 크롤링 + 하드코딩 (programs.ts)
- **가이드/레시피/사례**: Supabase DB (v4 앱과 공유)
- **운동 데이터**: exercises 하드코딩 (v4 앱 재사용)
- **배너 데이터**: localStorage (관리자 페이지에서 수정)
- **병원 정보**: 하드코딩 (주소, 전화, 진료시간)

## 6. 데이터 모델

### 6.1 Supabase 테이블

| 테이블 | 용도 | 주요 컬럼 |
|--------|------|----------|
| users | 사용자 | email, name, role (parent/admin), password |
| children | 자녀 | parent_id, name, gender, birth_date |
| measurements | 측정 기록 | child_id, measured_date, height, weight |
| daily_routines | 일일 루틴 | child_id, routine_date, sleep_time, wake_time |
| meals | 식사 | daily_routine_id, meal_type, description |
| meal_photos | 식사 사진 | meal_id, photo_url |
| meal_analyses | AI 분석 | meal_id, calories, protein, growth_score, advice |
| exercise_logs | 운동 | daily_routine_id, exercise_key, sets, reps |
| recipes | 레시피 | title, image_url, is_published |
| growth_guides | 가이드 | title, image_url, content, is_published |
| growth_cases | 사례 | patient_name, gender, measurements (JSON) |

### 6.2 Supabase Storage

| 버킷 | 용도 | 폴더 |
|-------|------|------|
| content-images | 콘텐츠 이미지 | recipes/, guides/, cases/, banners/ |
| meal-photos | 식단 사진 | (flat) |

### 6.3 클라이언트 저장소

| 키 | 저장소 | 용도 |
|----|--------|------|
| website-banners | localStorage | 배너 슬라이드 데이터 |
| website-admin-auth | sessionStorage | 배너 관리자 인증 상태 |
