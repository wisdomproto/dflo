# 광고 어드바이저 (Ad Advisor) 설계

- 작성일: 2026-06-19
- 위치: `/marketing/advisor`
- 한 줄: **목표(주간)를 넣으면 실측 데이터 기반으로 예산을 역산하고, 소재 유지/교체/신규 후보를 추천하며, 신규 광고 소재의 훅 스토리(태국어+한국어)를 생성하는 온디맨드 로보어드바이저.**

## 배경 / 목적

187 성장클리닉 태국 Meta 광고를 매 세션 처음부터 다시 논의하지 않고, **신호 → 추천**이 자동으로 도는 도구가 필요. 주식 로보어드바이저 비유: 데이터를 읽어 "이번 주 이렇게 굴려라"를 근거와 함께 출력하되, **Meta 앱이 개발자 모드라 자동 집행은 불가 → 추천까지** 하고 집행(Ads Manager 세팅)은 사람이.

확정 전략(`v4/public/marketing/strategy/th-launch-ads.html`, memory `marketing_strategy_thai_launch.md`)을 코드로 구현하는 운영 도구.

## 핵심 결정 (브레인스토밍 확정)

1. **데이터 게이트(B)**: 데이터가 충분할 때만 작동. 부족하면 "데이터 부족 + 무엇을 모아야 하는지" 안내. 신뢰 못 할 추천을 뱉지 않는다.
2. **목표 입력(B)**: 주간 ① 계산기 완료수(`height_calc_complete`) ② LINE 리드(`Lead`) ③ 영상 시청자 풀 증가. 단위 = **주**.
3. **소재 선별(C 하이브리드)**: 오가닉 실적 1순위 + 주제 적합도 휴리스틱 2순위.
4. **신규 스토리(B)**: 광고용 훅 스토리(훅 + 30초 아크 + 핵심 자막 카피), 태국어+한국어. 풀 제작은 기존 스토리보드 파이프라인이 받음.
5. **저장(A)**: 온디맨드 휘발성. DB 변경/migration 없음.
6. **구현(2안 하이브리드)**: 숫자(게이트·CPA·예산·스코어)는 코드 순수 함수(결정론적·테스트 가능), AI는 스토리 생성·문장 다듬기에만.

## 아키텍처 / 데이터 흐름

```
[/marketing/advisor]
  목표 입력 → POST /api/marketing/advisor/recommend
    1. 데이터 수집 (기존 서비스 재사용)
       metaAds insights · ga4SiteBreakdown · metaFeed · 릴스 자산 현황(62토픽)
    2. 데이터 게이트 판정 (코드, 지표별)
    3. 코드 엔진 (결정론적)
       실측 CPA·전환율 → 예산 역산 → 단계 판정/배분 → 소재 스코어링
    4. AI 레이어 (생성만)
       신규 소재 훅 스토리(태국어+한국어) + 추천 사유 다듬기
  → JSON 응답(저장 안 함) → 화면: 게이트 신호등 · 예산 · 소재 · 스토리
```

### 모듈 분리 (각각 독립 테스트)
ai-server `src/services/advisor/`:
- `advisorData.ts` — 데이터 수집·정규화(기존 서비스 호출 묶음, I/O)
- `advisorGate.ts` — 데이터 충분성 판정 (순수)
- `advisorBudget.ts` — 예산 역산·단계 배분·수익성 점검 (순수)
- `advisorCreative.ts` — 소재 스코어링·신규 후보 선별 (순수)
- `advisorStory.ts` — AI 훅 스토리 프롬프트 (생성)
- route: `routes/marketing.ts`에 `POST /advisor/recommend` 핸들러 추가

v4 `src/features/marketing/`:
- `components/AdAdvisorPage.tsx` — 페이지(목표 폼 + 결과 4섹션)
- `services/adAdvisorService.ts` — API 호출
- 사이드바 메뉴 "🤖 광고 어드바이저" + router lazy 등록

## 컴포넌트 상세

### 1. 데이터 게이트 (`advisorGate.ts`)
- 측정 윈도우: **직전 14일** (상수, 조정 가능).
- 지표별 통과 조건 (시작 기본값, 캘리브레이션 가능):

| 게이트 | 통과 조건 | 부족 메시지 |
|--------|----------|------------|
| 전역(광고) | 지출 발생 캠페인 ≥1 + 가동 ≥7일 | "아직 광고 성과 데이터 없음 — 먼저 캠페인 가동" |
| 계산기 완료 | 광고 기여 완료 ≥30건/윈도우 | "계산기 CPA 산출 불가 — N건 더 필요(현재 X)" |
| LINE 리드 | 광고 기여 리드 ≥30건/윈도우 | "LINE 전환 데이터 부족 — N건 더 필요" |
| 시청자 풀 | 동영상 조회 데이터 ≥7일 | "조회 데이터 부족 — 참여 광고 며칠 더" |
| 개별 소재 | 노출 ≥1,000 또는 가동 ≥3일 | 해당 소재만 "판단 보류" |

- 30건 = CPA 신뢰 최소선(Meta 학습 50/주의 보수적 하한). 상수.
- 출력: 지표별 신호등(🟢/🔴) + 부족 시 필요량·현황·예상 도달 시점. 부족 지표 카드만 잠금, 통과 지표는 정상 진행.
- **광고 기여 귀속의 한계**: GA4 소스/매체(paid) 분리 + Meta 픽셀 전환수 교차검증으로 근사. 화면에 "GA4 귀속 기준, ±오차" 명시.

### 2. 예산 역산 (`advisorBudget.ts`)
```
트랙 예산(주) = 목표 건수 × 실측 CPA(지표)   // 게이트 통과 지표만
총 주간 예산 = 트랙 합 ;  일예산 = 트랙 예산 ÷ 7
```
- **단계 가드레일**: 풀 크기·픽셀 성숙도로 단계(초기/중기/성숙) 판정 → 역산 배분 vs 단계 권장 비율(70/30→50/50→30/70) 비교. 어긋나면 경고("현재 풀 작아 LINE 목표 비현실 — 목표↓ 또는 풀 먼저 키우기"). 퍼널 의존성(리드는 풀 없이 싸게 안 나옴)을 현실 점검으로.
- **수익성 점검**: 역산 리드당 비용 > 흑자선(20~40만원)이면 🔴. 예상 매출 환산("리드 N → 치료 ~20% → 매출 추정", 가정 전환율 라벨링).
- **목표 일부만 입력**: 입력한 지표만 역산, 나머지 트랙은 단계 권장 비율 기반 "권장(목표 미설정)" 회색 표시.

### 3. 소재 추천 (`advisorCreative.ts`)
- **가동 광고 평가**: 트랙별 지표 종합 점수.
  - 참여: 3초 조회단가↓·시청유지율↑·CTR↑·빈도
  - 홈페이지: 완료당 비용↓·링크 CTR↑·랜딩 완료율↑
  - 판정: 🟢유지(상위+빈도<3.5) / 🔴컷(하위) / 🟡교체(빈도≥3.5) / ⚪보류(데이터 부족)
- **신규 후보(하이브리드)**: 1순위 오가닉 실적(metaFeed, 미광고 릴 도달·참여 상위) → 2순위 주제 적합도(62토픽 AI 점수 = 훅 강도·페인 매칭·중복 제외·태국어 자산 여부).
- 출력: 유지/컷/교체/🆕신규 후보 리스트(점수·사유·태국어 자산 상태).

### 4. 신규 스토리 (`advisorStory.ts`)
- 트리거: 🔁교체 대상 + 🆕후보 중 태국어 릴 미보유.
- 구조: 훅(첫 3초) / 내러티브 아크(도입→문제→반전→CTA) / 핵심 자막 카피 3~4줄(태국어+한국어) / CTA(트랙별) / 변형 베이스(릴#N).
- 제약 내장: 의료광고 규정(효과보장·최고/유일 금지·개인차 명시) + 남성 의사 격식체(태국어 ครับ/ผม, 여성형 금지).
- 출력: 복사 가능 카드. 풀 제작은 기존 스토리보드 파이프라인(추후 "스토리보드로 보내기").

### 5. UI (`AdAdvisorPage.tsx`)
- 상단: 목표 폼(계산기 완료·LINE 리드·시청자 풀, 측정 윈도우 표시) + [지금 추천받기].
- 결과 4섹션: ① 게이트 신호등 ② 예산 카드(총/일예산·트랙 배분·예상성과·경고) ③ 소재 추천 ④ 신규 스토리.
- 데이터 부족 시: ②③④ 잠금 + "부족 패널"(필요량·현황·먼저 할 일)만.

## 운영 제약
- `/api/marketing/advisor/*`는 다른 마케팅 라우트처럼 prod auth 게이트 → **현재 dev(localhost) 전용**. GA4/analytics(`/api/analytics/*`)는 무인증이라 prod서도 동작.
- Gemini 키 필요(스토리 생성). 키 없으면 게이트·예산·소재 추천은 동작, 스토리만 graceful skip.
- 저장 없음 → migration 없음.

## 테스트
- 순수 함수(`advisorGate`/`advisorBudget`/`advisorCreative`) 단위 테스트: 게이트 경계(29 vs 30건), 예산 역산 산수, 단계 가드레일 분기, 수익성 경고선, 소재 판정 규칙(빈도 3.5 경계), 목표 일부 입력.
- `advisorData`는 외부 I/O라 통합 테스트/모킹.

## YAGNI / 비범위
- 추천 히스토리 저장·주차별 비교(추후 B 옵션).
- Meta 자동 집행(앱 라이브·검수 후).
- 풀 스토리보드 생성(기존 파이프라인 재사용).

## 미정 (런칭 후 캘리브레이션)
- 게이트 임계치(14일/30건), 빈도 교체선(3.5), 가정 전환율(20%) — 첫 2~4주 실데이터로 조정. 전부 상수로 노출.
