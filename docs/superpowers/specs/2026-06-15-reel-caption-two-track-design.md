# 릴스 자막 2트랙 시스템 — 설계

날짜: 2026-06-15 · 상태: 승인됨(설계) → 구현 계획 대기

## 목적

태국 FB/IG 타겟 릴스의 화면 텍스트를 **두 개의 목적이 다른 레이어**로 분리한다.

1. **강조(emphasis)** — 크게 박히는 핵심 문구·숫자. 후킹·리텐션용. 간헐 등장(구두점).
2. **이해용 자막(caption)** — 풀 나레이션을 말에 맞춰 한 구절씩 보여주는 카라오케 자막. 음소거 시청·태국 부모의 이해 채널.

현재는 단일 트랙뿐이다: `cap_{lang}`(짧은 요약 2줄)이 하단(1404px)에 큰 글씨로 뜨는 게 사실상 "강조" 역할이고, **풀 나레이션이 들어갈 이해용 자막 트랙이 아예 없다.** 또 자막 타이밍이 **청크 단위(5~7초에 캡션 하나)** 라 구절 단위 동기가 불가능하다.

## 범위

- **대상**: `remotion/` 의 `PresenterShort` 렌더러 + `script.json` 스키마 + 빌드 스크립트(구절 타이밍 생성).
- **제외**:
  - 서버 워커·발행 파이프라인(앞서 논의한 GPU 립싱크 워커는 별개 스펙).
  - 더빙 vs 자막 노선 결정(이 시스템은 양쪽을 모두 지원하도록 언어-중립 설계 — 노선은 렌더 설정에서 선택).
- **적용 시점**: 신규 릴스부터 표준. 기존 제작분(예: #2·#3 ko/th)은 재렌더가 필요하므로 **선택적 소급**(원하는 것만 나중에 재렌더).

## 핵심 결정

| 항목 | 결정 |
|---|---|
| 이해 자막 단위 | **구절(카라오케)**. 청크 안을 글자수 비례로 균등 분할 |
| 강조 위치 | **상단 오버레이(A안)** — 영상 위 floating, 핵심 청크에만 pop→hold→fade |
| 인서트 구간 강조 | **자동 숨김** — 인포그래픽 + 자체 라벨이 강조 역할. 하단 자막은 계속 |
| 언어 | 출력본당 **단일 언어 통일**(강조·자막 둘 다 같은 언어). 오디오 언어는 별개 선택 가능 |
| 자막 텍스트 소스 | 기존 `{lang}` 풀 나레이션 **자동 분할**(새 authoring 없음) |
| 강조 텍스트 소스 | 기존 `cap_{lang}` 요약을 `emph_{lang}` 로 이관 |

## 데이터 두 트랙 (script.json)

### 강조 트랙
- 청크별 `emph_{lang}`: 짧은 1~2줄 배열(현재 `cap_{lang}` 내용 이관).
- **핵심 청크에만 채움**, 나머지는 비움(빈 배열/필드 없음) → 간헐 등장으로 강조 효과 유지.
- `hl_{lang}`(강조 내 노란색 하이라이트 구절)는 강조 트랙에 그대로 따라감.

### 이해 자막 트랙
- 청크별 `{lang}` 풀 나레이션 텍스트(이미 존재 — TTS 생성 소스)를 **빌드 시 구절 분할**.
- 한 출력본은 언어 L 하나로 통일: 강조·자막 모두 `…_{L}` / `{L}` 사용.
- **오디오 독립성**: 오디오 언어 A 는 텍스트 언어 L 과 달라도 됨.
  - 태국어 더빙: L=th, A=th(태국어 립싱크 음성).
  - 한국어 음성 + 태국어 자막: L=th(강조·자막 태국어), A=ko(한국어 음성/영상).
  - 렌더 설정(컴포지션 prop)에서 텍스트 언어와 오디오/영상 소스를 각각 지정. 스키마는 언어별 텍스트만 보유.

### 구절 분할 규칙
- 공백·문장부호 기준으로 토큰화 → 2~7단어(또는 태국어는 transcreation 문장의 기존 띄어쓰기=구절 경계) 단위로 묶음.
  - 태국어는 단어 사이 공백이 없지만 **구절·절 사이엔 공백**이 있으므로 공백 분할이 구절 경계로 동작.
- 각 구절에 청크 `durFrames` 를 **글자수 비례로 배분**(구절 durFrames 합 = 청크 durFrames).
- ElevenLabs 문자 타임스탬프는 v1에서 불필요(비례 분할로 충분 — 자막은 번역이라 어차피 음성과 1:1 단어 대응이 아님).

## 레이아웃 기하 + 세이프존 (1080×1920)

```
0–300       헤더 (로고·훅)
300–1380    원장 영상 / 인포그래픽 인서트  (PANEL_TOP=300, PANEL_H=1080)
  └ 강조 오버레이: 영상 상단(헤더 바로 아래)에 floating
                    핵심 청크에만 pop→hold→fade
                    ★ 인서트 청크면 렌더 skip
~1392–1596  이해 자막 밴드 (카라오케 구절, 최대 2줄)  ← 세이프존 위로 상향
1596–1920   로고 워터마크 (하단 일부는 플랫폼 UI 영역 → 비핵심만)
```

- **세이프존 수정**: 현재 자막이 `CAP_TOP=1404, CAP_H=280`(하단 1684=87.7%)라 하단부가 IG/FB/TikTok UI(계정명·액션버튼·진행바)에 가려진다. → 자막 밴드를 **약 1392~1596**(하단 ~83% 이내)으로 올려 세이프존 확보. 정확값은 실제 플랫폼 오버레이 캡처로 미세조정.
- `Thumb.tsx` 의 고정 Y-슬롯 패턴을 따라 **고정 밴드 상수**로 박아 위치가 튀지 않게.
- 중앙 원장 얼굴 영역은 강조/자막 어느 쪽도 침범하지 않음(신뢰=얼굴).

## 렌더러 컴포넌트 (PresenterShort 확장)

`remotion/src/shorts/_shared/PresenterShort.tsx`:

- **`EmphasisZone`(신규)**: 상단 오버레이 레이어.
  - `emph_{lang}` 가 있는 청크에만 렌더, 인서트 청크(`insert`/`insert_{lang}` 존재)면 skip.
  - 등장 애니메이션 pop-in → hold → fade-out(청크 경계).
  - `hlLine()` 하이라이트 재사용.
- **`CaptionZone`(개편)**: 하단 세이프존 밴드.
  - 기존 "청크당 단일 블록" → **구절별 `<Sequence from/durationInFrames>`** 카라오케 등장.
  - `captions-{lang}.json` 의 청크별 구절 배열을 소비. 청크 시작 프레임(FROM[i]) + 구절 누적 오프셋으로 각 구절 Sequence 배치.
  - 파일 없으면 폴백: 기존처럼 청크 풀 나레이션 1블록(무회귀).
- `InsertPanel`·오디오 Sequence 동기는 **불변**. 강조/자막은 독립 레이어라 인서트와 토글만 연동.
- 레이아웃 상수 분리: `PANEL_TOP/PANEL_H/CAP_TOP/CAP_H` 등을 명시 상수로 두고 자막 밴드 Y를 세이프존 값으로 교체.

## 빌드 파이프라인

- **신규 `remotion/scripts/gen-captions.mjs`**:
  - 입력: `src/shorts/{slug}/script.json`(청크별 `{lang}` 나레이션) + `timing-{lang}.json`(청크 durFrames).
  - 처리: 순수 함수 `splitIntoPhrases(text, lang)` + `distributeFrames(phrases, durFrames)`.
  - 출력: `src/shorts/{slug}/captions-{lang}.json` — `{ [chunkId]: [{ text, fromFrame, durFrames }] }`.
  - 수동 구절경계 보정 가능(재실행 안전, 기존 수동본 보존 옵션은 추후).
- CaptionZone 이 이 파일을 import. 누락 시 graceful 폴백.

## 테스트

- `gen-captions` 순수 함수 단위테스트(`node --test`):
  - 한국어 공백 분할 → 2~7단어 묶음.
  - 태국어 구절-공백 분할(단어 내 미분할).
  - 비례 타이밍: 구절 durFrames 합 = 청크 durFrames(반올림 오차 마지막 구절 흡수).
  - 빈/짧은 나레이션 엣지(1구절).
- Remotion 시각 검증: `npx remotion still`(또는 `--frames`)로 강조-청크/인서트-청크/자막 구절 각 1프레임 추출 → 3레이어가 의도한 Y밴드에 오는지, 인서트 청크에서 강조가 숨는지 확인.

## 마이그레이션 / 무회귀

- 기존 `cap_{lang}` → `emph_{lang}` 이관: 일회성 스크립트(또는 수동, 컷 수가 적으면). 이관 전엔 `EmphasisZone` 이 `emph_{lang}` 없으면 `cap_{lang}` 폴백 → 점진 이관 안전.
- `captions-{lang}.json` 없는 기존 컷: CaptionZone 이 기존 단일블록으로 폴백 → 재렌더 전까지 무회귀.
- 신규 컷부터 2트랙 표준 적용.

## 열린 항목(구현 중 확정)

- 자막 밴드 정확 Y/높이: 실제 IG·FB 릴스 UI 오버레이 캡처 후 픽셀 확정.
- 구절 최대 길이(단어수/글자수) 튜닝값.
- 강조 hold 시간(프레임) 튜닝값.
